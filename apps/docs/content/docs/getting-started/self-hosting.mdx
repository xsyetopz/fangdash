<Callout type="info">
	**TL;DR** - FangDash runs entirely on Cloudflare infrastructure (Workers + D1) - Three services to
	deploy: API, Web, and PartyKit (multiplayer) - `bun run ship` deploys all three at once
</Callout>

FangDash is designed to run on Cloudflare's infrastructure. Here's how to deploy your own instance.

## What You Need

| Requirement                  | Notes                     |
| ---------------------------- | ------------------------- |
| Cloudflare account           | Free tier works           |
| Twitch Developer application | For OAuth sign-in         |
| PartyKit account             | For real-time multiplayer |

<Steps>
<Step>
## Deploy the API

The **API** runs on Cloudflare Workers with a D1 database.

1. Create a D1 database in the Cloudflare dashboard
2. Update `apps/api/wrangler.toml` with your database ID and your production domains:

```toml
[env.production.vars]
ENVIRONMENT = "production"
BETTER_AUTH_URL = "https://your-api-domain.workers.dev"
WEB_URL = "https://your-web-domain.workers.dev"
```

3. Set your secrets:

```bash
bunx wrangler secret put BETTER_AUTH_SECRET --env production
bunx wrangler secret put TWITCH_CLIENT_ID --env production
bunx wrangler secret put TWITCH_CLIENT_SECRET --env production
```

4. Optionally, auto-promote specific Twitch users to admin on first login:

```bash
bunx wrangler secret put ADMIN_TWITCH_ID --env production
# Value: comma-separated Twitch numeric user IDs, e.g. 123456789,987654321
```

5. Deploy:

```bash
bun run ship:api
```

</Step>

<Step>
## Deploy the Web App

The **web app** deploys to Cloudflare Workers using OpenNext.

1. Set your API URL as a build-time environment variable:

```bash
NEXT_PUBLIC_API_URL=https://your-api-domain.workers.dev bun run ship:web
```

Or add it to `apps/web/.env.local` first:

```bash
NEXT_PUBLIC_API_URL=https://your-api-domain.workers.dev
NEXT_PUBLIC_PARTYKIT_HOST=fangdash.your-username.partykit.dev
```

2. Deploy:

```bash
bun run ship:web
```

</Step>

<Step>
## Deploy PartyKit

**PartyKit** handles real-time multiplayer WebSocket connections.

1. Set your PartyKit token:

```bash
export PARTYKIT_TOKEN=your-token
```

2. Deploy:

```bash
bun run ship:party
```

</Step>
</Steps>

## Deploy Everything at Once

```bash
bun run ship
```

This runs all three deploy commands in parallel.

## Environment Variables Reference

### API (`apps/api`)

| Variable               | Type      | Where set                        | Description                                         |
| ---------------------- | --------- | -------------------------------- | --------------------------------------------------- |
| `BETTER_AUTH_SECRET`   | Secret    | `wrangler secret put`            | Long random string for signing sessions             |
| `TWITCH_CLIENT_ID`     | Secret    | `wrangler secret put`            | Twitch OAuth app client ID                          |
| `TWITCH_CLIENT_SECRET` | Secret    | `wrangler secret put`            | Twitch OAuth app client secret                      |
| `ADMIN_TWITCH_ID`      | Secret    | `wrangler secret put` (optional) | Comma-separated Twitch user IDs to auto-admin       |
| `BETTER_AUTH_URL`      | Plaintext | `wrangler.toml`                  | Public URL of the API worker                        |
| `WEB_URL`              | Plaintext | `wrangler.toml`                  | Public URL of the web worker (for CORS + redirects) |
| `ENVIRONMENT`          | Plaintext | `wrangler.toml`                  | `"development"` or `"production"`                   |

### Web (`apps/web`)

| Variable                    | Type      | Where set             | Description                                       |
| --------------------------- | --------- | --------------------- | ------------------------------------------------- |
| `NEXT_PUBLIC_API_URL`       | Plaintext | `.env.local` / CI env | Public URL of the API worker                      |
| `NEXT_PUBLIC_PARTYKIT_HOST` | Plaintext | `.env.local` / CI env | PartyKit host (e.g. `fangdash.user.partykit.dev`) |

### Local Development

Copy the example files and fill in your values:

```bash
cp apps/api/.dev.vars.example apps/api/.dev.vars
cp apps/web/.env.example apps/web/.env.local
```

## Setting Up Twitch OAuth

<Steps>
	<Step>Go to the [Twitch Developer Console](https://dev.twitch.tv/console)</Step>
	<Step>Create a new application</Step>
	<Step>Set the OAuth Redirect URL to `https://your-api-domain/api/auth/callback/twitch`</Step>
	<Step>Copy the Client ID and Client Secret to your environment variables</Step>
</Steps>

<Callout type="warn">
	The Twitch OAuth redirect URL must match your deployed API domain exactly — including the path
	`/api/auth/callback/twitch`.
</Callout>
